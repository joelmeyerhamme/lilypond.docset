2.1.1 Spécificités pour MacOS X

Utilisation de scripts Python sur MacOS 10.3 ou 10.4

Les exécutables de LilyPond pour MacOS X ne fournissent pas Python, bien qu’une version au moins de niveau 2.4 soit requise par convert-ly. Ainsi, et si vous utilisez MacOS 10.3 ou 10.4, devrez-vous installer une version plus récente de Python à partir de http://python.org/download/, puis éditer la première ligne de convert-ly et lilypond-book. Si les exécutables du Python que vous venez d’installer sont dans votre PATH, cette ligne doit être : 

#!/usr/bin/env python

et dans le cas contraire 

#!/chemin/vers/nouvelle_installation/python

MacOS X et la ligne de commande

Les scripts tels que lilypond-book, convert-ly, abc2ly, et même lilypond, sont insclus dans un fichier .app pour MacOS X. Vous pourrez les lancer directement en ligne de commande de la manière suivante : 

chemin/vers/LilyPond.app/Contents/Resources/bin/lilypond

Il en va de même pour les autres scripts de ce répertoire, y compris lilypond-book, convert-ly, abc2ly, etc.

Une autre façon de procéder consiste à créer des scripts qui ajoutent automatiquement le chemin. À cette intention, créez un répertoire qui contiendra ces scripts : 

mkdir -p ~/bin
cd ~/bin

Créez un fichier appelé lilypond et qui contiendra : 

exec chemin/vers/LilyPond.app/Contents/Resources/bin/lilypond "$@"

Créez autant de fichiers – lilypond-book, convert-ly, et autres programmes annexes que vous utilisez (abc2ly, midi2ly, etc) – que de besoin. Remplacez simplement bin/lilypond par bin/convert-ly ou tout autre nom de programme dans le fichier que vous créez.

Rendez ces fichiers exécutables : 

chmod u+x lilypond

Ajoutez alors ce répertoire à votre PATH. Modifiez le fichier .profile – créez-le si besoin – de votre répertoire personnel, de telle sorte qu’il contienne 

export PATH=$PATH:~/bin

Ce fichier doit se terminer par une ligne vide.

Notez que chemin/vers devrait correspondre à /Applications/.
